==============================================================================
PART B — AI MODELS (DOWNLOADED AT RUNTIME, NOT BUNDLED)
==============================================================================

AMAdocs ships with NO AI model. On first run the user downloads one. Only
permissively-licensed models are offered (the in-app catalog is an MIT/Apache
allowlist). Listed here for transparency:

  • Phi-3.5-mini   — License: MIT        (Microsoft)   [default chat model]
  • Phi-4-mini     — License: MIT        (Microsoft)
  • Qwen3 (1.7b/4b)— License: Apache-2.0 (Alibaba)
  • Mistral        — License: Apache-2.0 (Mistral AI)
  • Moondream      — License: Apache-2.0 (vision / image understanding)

Each model's license travels with the model when downloaded by Ollama.

The following notices cover all npm packages bundled in the engine (server +

  • Dual-licensed packages are used under their permissive option:
      jszip (MIT OR GPL-3.0)            -> MIT
      @zone-eu/mailsplit (MIT OR EUPL)  -> MIT
      dompurify (MPL-2.0 OR Apache-2.0) -> Apache-2.0
  • web-push is MPL-2.0 (file-level copyleft; source at its repository).
  • A few legacy packages declare no SPDX id or license file:
      buffers (substack) and combine-errors (Matthew Mueller) follow their
      authors' standard MIT licensing; voyageai is the Voyage AI TypeScript
      SDK (https://github.com/voyage-ai/typescript-sdk). These, along with
      some optional provider SDKs (cohere-ai, @zilliz/milvus2-sdk-node), are
      transitive dependencies of the upstream engine that AMAdocs does not
      invoke (AMAdocs uses Ollama + the native embedder + LanceDB).
